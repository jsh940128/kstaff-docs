# K-Staff 작동 방식

이 페이지는 K-Staff의 전체 라이프사이클을 한 페이지에 담은 가이드입니다. 4가지 역할이 어떤 순서로 무엇을 하는지, 어디서 인계(hand-off)가 일어나는지 보여줍니다. 각 단계는 의도적으로 간결하게 작성되어 있으며, 자세한 화면과 절차는 [역할별 가이드](/docs/overseas-partner)에서 확인할 수 있습니다.

---

## 인계 흐름도

```
해외 파트너            플랫폼 관리자            고용주                       한국 파트너
---------            ----------            ------                       ---------
후보자 등록
  (Tier-1/2/3) ...> 공급 가시화
서류 업로드

                    MSA 계약 게시
                    고용주 계정 생성 .....> 계정 생성됨
                                           MSA 서명
                    신뢰 등급 상향 <........ 컴플라이언스 대시보드
                                           이력서 접근 요청
                    Tier-2 권한 부여 .....> Tier-2 이력서 조회
                                           수요 요청 등록
                    상태: 검토 중
                    한국 파트너로 라우팅 ...........................................> 수요 리스트 수신
                                                                                     리스트 수락
                                                                                     Tier-1만 조회
                                                                                     (이력서/문서 잠금)
                                                                                     부적합 거절
                                                                                     면접 요청
                    면접 승인 <................................................... (대기 중)
                                                                                     후보자 잠금
                                                                                     (수수료 동의;
                                                                                      Tier-2 + Tier-3
                                                                                      열림)
                                                                                     채용 체결 보고
                    수수료 제안 ...............................................> 수수료 확인
                    채용 확정
                    계약서 초안 생성
                    계약서 최종 확정 ......> 고용주 최종본 조회
후보자
"체결 완료" 표시     E-7-4 점수 + 마감 <................................... E-7-4 프로필 입력
```

채용 확정 이후에도 관계는 지속됩니다. E-7-4 비자 업그레이드 점수 산정, 컴플라이언스 마감 알림(D-7 / D-1), 30일 · 1년 · 3년 마일스톤 알림, 계약 만료 시점의 재고용 가능자 추적이 자동으로 이루어집니다.

## 라이프사이클 (순서대로)

### 1. 공급: 후보자가 시스템에 진입

**해외 파트너**가 후보자를 등록합니다. 한 번의 제출로 3단계 정보가 모두 입력됩니다.

- **Tier-1 (공개 정보):** 국가, 직무 카테고리, 프로필 요약.
- **Tier-2 (이력서):** 학력, 경력, 자격증.
- **Tier-3 (문서):** 여권 등 민감 파일 (Cloudflare R2에 업로드).

이 시점부터 후보자가 시스템에 존재하며, 접근 권한이 부여되기 전까지 다른 역할들이 후보자를 식별할 때 사용할 익명 `public_code`가 부여됩니다.

### 2. 컴플라이언스: 고용주 온보딩

**플랫폼 관리자**가 현재 유효한 **마스터 서비스 계약(MSA)** 을 게시하고 고용주 계정을 프로비저닝합니다. 고용주는 로그인 후 컴플라이언스 대시보드를 만나게 됩니다. MSA에 서명하기 전까지 **모든 Tier-2 엔드포인트는 403을 반환합니다**. 약관 서명 없이는 이력서 열람이 불가능합니다.

### 3. 신뢰 및 의도: Tier-2 잠금 해제

MSA 서명 후, 고용주는 Tier-1 후보자를 검색할 수 있습니다. 특정 후보자의 이력서를 열기 위해서는 다음 조건을 모두 충족해야 합니다.

1. **신뢰 등급**이 충분해야 합니다. 플랫폼 관리자가 관계에 따라 `restricted`에서 `verified`, `trusted`, `partner`로 승격합니다.
2. 해당 후보자에 대해 **이력서 접근 요청**을 제출해야 합니다.
3. 플랫폼 관리자가 `(고용주, 후보자)` 쌍에 대해 Tier-2 접근 권한을 부여해야 합니다.

접근 권한은 약 **365일** 동안 유효하며, 재발급 시 동일 권한이 갱신되므로 채용 진행 중에 권한이 갑자기 만료되지 않습니다. `trusted` 등급의 고용주라도 후보자별로 명시적 권한이 필요합니다.

### 4. 수요: 고용주 요청 등록

고용주가 직무, 인원수, 요구사항, 관심 후보자의 `public_code`를 포함한 **수요 요청(demand request)** 을 제출합니다. 요청은 `open` 상태로 플랫폼에 진입합니다.

### 5. 검토 및 라우팅

플랫폼 관리자가 요청을 검토하고(`status = reviewing`), 적합한 한국 파트너에게 **라우팅**합니다. 라우팅 시점에 해당 한국 파트너의 대시보드에 표시되는 **수요 리스트(demand list)** 가 생성됩니다. 플랫폼 관리자는 검토를 위해 모든 후보자의 전체 프로필과 문서에 접근할 수 있습니다.

### 6. 한국 파트너 실행 (방화벽)

한국 파트너가 수요 리스트를 수락합니다. **잠금 이전까지 파트너는 Tier-1 정보만 볼 수 있습니다.** 이력서나 문서 열람 시도는 `403`을 반환합니다. 이 단계에서 파트너는 다음을 수행합니다.

- 적합하지 않은 후보자 거절.
- 면접 요청 (플랫폼 관리자의 승인 필요).

채용 의사가 확정되면 후보자를 **잠금(lock)** 합니다. 잠금은 시스템에서 가장 중요한 게이트로, 동시에 두 가지 효과를 가집니다. **(a)** 해당 파트너에게 Tier-2 (이력서)와 Tier-3 (문서)을 열어줍니다. **(b)** 파트너가 수수료에 대해 약정한 것으로 간주됩니다. 잠금 이후에는 실제 채용 진행에 필요한 모든 정보에 접근할 수 있습니다.

### 7. 채용 체결

한국 파트너가 근로자를 채용한 후, 계약 시작일을 포함한 채용 체결(placement)을 보고합니다. 플랫폼이 **2단계 수수료 핸드셰이크**를 진행합니다.

1. 플랫폼 관리자가 최종 수수료를 **제안**합니다.
2. 한국 파트너가 수수료를 **확인**합니다.
3. 플랫폼 관리자가 채용 체결을 **확정**합니다.

후보자 상태는 `placed`가 되고, 고용주 대시보드에 표시됩니다. 해외 파트너에게는 익명화된 단계(`Placed`)만 노출되며, 고용주명이나 수수료는 보이지 않습니다.

### 8. 계약서

플랫폼 관리자가 **이중 언어(한국어/영어) 계약서 초안**을 생성합니다. 초안은 재생성(각 회차마다 버전 증가)이 가능하고 내부 검토 후 수정할 수 있습니다. **고용주는 초안을 볼 수 없습니다.** 플랫폼 관리자가 **최종 확정(finalize)** 한 계약서만 고용주에게 표시됩니다.

### 9. 채용 이후: 지속적 컴플라이언스

채용 체결 이후에도 K-Staff는 사례를 지속적으로 관찰합니다.

- **마감 알림.** `compliance_deadlines` 항목에 대해 D-7 및 D-1 이메일 알림이 고용주에게 발송됩니다.
- **마일스톤 알림.** 계약 시작일 기준 30일, 1년, 3년 시점에 자동 알림이 발송됩니다.
- **E-7-4 자격 점수.** 한국 파트너와 플랫폼 관리자가 TOPIK, 학력, 자격증 정보를 입력하면 점수가 자동 재계산됩니다.
- **재고용 추적.** 계약 만료가 다가오면 고용주의 `rehire-eligible` 목록에 후보자가 표시됩니다.

## K-Staff의 차별점

위 흐름에서 보이는 3가지 특징이 있습니다.

1. **잠금은 실제 경계선입니다.** 단순한 체크박스가 아닙니다. 한국 파트너는 채용 의사를 확정하기 전까지 이력서와 문서를 볼 수 없습니다. 이는 시스템에서 가장 강력한 게이트입니다.
2. **Tier-2 접근은 후보자별, 시간 제한, 운영자 발급입니다.** `trusted` 등급의 고용주라도 이력서를 일괄 수집할 수 없습니다. 플랫폼 관리자가 특정 후보자에 대해 권한을 발급하며, 권한은 만료됩니다.
3. **모든 민감한 단계는 운영자가 중개하고 감사 기록됩니다.** 면접 승인, 수수료 제안, 채용 확정, 계약 최종 확정은 모두 플랫폼 관리자를 거치며 감사 로그에 기록됩니다.

실제 클릭 단위의 화면 설명을 원하시면 [역할별 가이드](/docs/overseas-partner)를 참고하세요.
